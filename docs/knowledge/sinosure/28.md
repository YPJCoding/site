# 你和产品经理、业务、测试对话时常见黑话

## 一句话先懂

很多“听不懂”不是因为你不会前端，而是对方默认你已经懂业务缩写和黑话。

## 高频黑话翻译

| 黑话 | 先翻译成人话 |
| --- | --- |
| 走限额 | 去给某个买方申请/调整信用限额 |
| 挂保单 | 让这笔业务归到某张保单责任下面 |
| 走可损 | 提交可能损失通知 |
| 走索赔 | 正式发起赔款申请 |
| 这个案子立了 | 已进入案件流程 |
| 先补件 | 材料不够，要继续上传 |
| 限额失效了 | 这个买方当前额度不能用了 |
| 这票出运 | 这一笔具体发货/交付记录 |
| 先做减损 | 先想办法别让损失继续扩大 |
| 赔后追偿 | 先赔，再继续追回欠款 |

## 一个非常实用的方法

对方说一句黑话时，你就在脑子里强制翻译成：

`对象 + 动作 + 状态`

例如：

“这个买方限额失效了，先别走这票出运。”

你就翻译成：

- 对象：买方限额、出运
- 动作：暂停处理
- 状态：限额不可用
