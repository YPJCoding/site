# 常见模块名词和业务动作映射

## 用法说明

这篇不是内部菜单真表，而是帮助你在看原型、菜单、需求标题时，能快速猜到“这大概在干什么”。

## 常见模块名词映射

| 模块名词 | 你可以先怎么理解 | 对应业务动作 |
| --- | --- | --- |
| 在线询保 | 用户先咨询或了解保障方案 | 询问、引导、转化 |
| 资信红绿灯 | 对买方或风险情况做快速信号提示 | 风险筛查 |
| 资信导航仪 | 基于商品/市场寻找客户和市场线索 | 市场开拓 |
| 资信报警器 | 对国家/行业/企业风险做持续监控 | 风险预警 |
| 国别全景 | 看某国整体情况 | 国别研究 |
| 行业概览 | 看某行业大盘 | 行业研究 |
| 项目分析 | 看项目或机会的综合信息 | 前期评估 |
| 保单查询 | 看已有保单状态 | 在保业务查询 |
| 限额查询 | 看买方额度是否可用 | 风险控制 |
| 出运统计/出运查询 | 看已经发货或交付的业务 | 责任跟踪 |
| 逾期分析 | 看哪些应收账款到期未收回 | 风险跟踪 |
| 报损/可损通知 | 告诉中国信保出风险了 | 风险报案 |
| 索赔申请 | 正式申请赔款 | 案件发起 |
| 单证上传 | 上传合同、发票、物流、催收等材料 | 证据提交 |
| 理赔进度 | 看案件处理到哪一步 | 案件查询 |
| 保单融资 | 基于保单做融资增信 | 融资协同 |

## 你读菜单时的一个实用方法

看到模块名时，不要先问“这个页面长什么样”，先问：

1. 它服务交易前、交易中还是交易后？
2. 它是“看”还是“办”？
3. 它围绕的是保单、买方、限额、出运还是案件？

这样很多名字就不玄学了。

## 一个例子

如果你在原型里同时看到：

- `资信红绿灯`
- `限额申请`
- `索赔申请`

你就可以快速判断：

- 第一个偏前置风控
- 第二个偏交易前/交易中的风险控制
- 第三个偏交易后的案件处理

## 高概率推断说明

这篇映射大量使用了中国信保公开材料中的功能命名和常见业务系统理解方式，因此适合“快速入门”。如果你后续拿到内部菜单，以内部菜单为准，再把这篇当翻译表修订即可。
